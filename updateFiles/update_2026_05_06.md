# 2026-05-06 更新：JSON 导入功能 + 导出优化

## 概述

本次更新实现了从 JSON 反向创建场景节点的导入功能，并优化了导出器（文件名加场景名、补全遗漏字段）。现在导出和导入形成闭环，方便跨场景或跨设备同步节点数据。

## 功能详解

### 一、导出：文件名包含场景名

**NodeJsonExporter.cs**

输出路径从固定字符串改为属性，自动取当前打开场景的名称：

```csharp
// 修改前
private const string OutputPath = "Assets/StreamingAssets/nodes.json";

// 修改后
private static string OutputPath =>
    $"Assets/StreamingAssets/nodes.{EditorSceneManager.GetActiveScene().name}.json";
```

例：场景名为 `LowerCampus` → 输出 `nodes.LowerCampus.json`；场景名为 `UpperCampus` → 输出 `nodes.UpperCampus.json`。

这样不同场景的节点数据不会互相覆盖。

---

### 二、导出：补全 `autoDismissDuration`

5/3 更新在 `NodeData` 和 `NodeComponent` 中添加了 `autoDismissDuration`，但 `NodeJsonDto` 漏掉了，导致导出的 JSON 不包含该字段。现已修复：

- `NodeJsonDto` 新增 `public float autoDismissDuration;`
- `ToDto()` 中赋值：`autoDismissDuration = comp.autoDismissDuration`

导出的 JSON 现在包含完整数据：

```json
{
  "id": "path_test",
  "isVisible": false,
  "autoDismissDuration": 5.0,
  ...
}
```

---

### 三、DTO 类重构

DTO 类原本嵌套在 `NodeJsonExporter` 内部（`private class`），导致 importer 无法引用。现在提取为文件作用域的 `internal class`：

```
NodeJsonExporter.cs
├── NodeJsonExporter (static class)     ← 导出逻辑
├── NodeListWrapper                     ← DTO，internal，importer 可共用
├── NodeJsonDto
├── Vec3Json
├── ConnectedNodesWrapper / ConnectedNodeDto
├── BilingualTextWrapper / TextEntry
├── ImageEntry
├── BilingualAudioWrapper / AudioEntry
└── CameraOverrideDto
```

importer 直接复用这些 DTO 类，无需重复定义。

### 四、新建：NodeJsonImporter

**NodeJsonImporter.cs** — 菜单：**Tools > Tour System > Import Nodes from JSON**

#### 完整流程

```
用户点击菜单
  → 弹窗选择 JSON 文件（默认定位 StreamingAssets/）
  → 读取并 JsonUtility.FromJson 解析
  → 对比场景中已有的 NodeComponent ID，检测冲突
  → 如有冲突，弹窗三选一：
      "跳过已存在" — 保留旧节点，只创建不冲突的
      "取消"       — 中止
      "替换已存在" — 删除旧节点，用 JSON 数据重建
  → 确保场景中有 "Nodes" 空父对象（无则创建）
  → 遍历每个节点：
      ├─ CreatePrimitive(Sphere) 创建球体
      ├─ 命名 = node.id
      ├─ 设置 Transform (position / rotation / scale)
      ├─ 移除 SphereCollider（不需要物理碰撞）
      ├─ 材质：可见节点 = 绿色，不可见节点 = 蓝色
      ├─ AddComponent<NodeComponent>
      └─ 填入全部字段
  → 选中所有新建节点
  → 标记场景 dirty + Console 输出摘要
```

#### 球体视觉效果

| 节点类型 | 颜色 | 大小 |
|----------|------|------|
| 可见节点（isVisible=true） | 绿色 (0.3, 0.8, 0.3) | 0.3 |
| 不可见节点（isVisible=false） | 蓝色 (0.3, 0.5, 0.9) | 0.2 |

NodeComponent 自带的 Gizmos 线框（触发半径球、连接线）仍然正常绘制。

#### 字段映射

importer 填充 NodeComponent 的全部字段：

| NodeComponent 字段 | JSON 来源 | 默认值 |
|-------------------|-----------|--------|
| `id` | `node.id` | — |
| `displayName` | `node.displayName` | — |
| `shortDescription` | `node.shortDescription` | — |
| `campusLocation` | `node.campusLocation` | `"lower"` |
| `isStartingNode` | `node.isStartingNode` | false |
| `isVisible` | `node.isVisible` | true |
| `radius` | `node.radius` | 5 |
| `autoDismissDuration` | `node.autoDismissDuration` | 0 |
| `thumbnail` | `node.thumbnail` | — |
| `zhText / enText` | `node.text.zh / node.text.en` | — |
| `images[]` | `node.images[]` | — |
| `zhAudio / enAudio` | `node.audio.zh / node.audio.en` | — |
| `connectedTo[]` | `node.connectedNodes.to[]` | — |
| `cameraOverride` | `node.cameraOverride` | — |

#### Undo 支持

导入操作支持 Ctrl+Z 撤销。所有创建的 GameObject 和 NodeComponent 均通过 `Undo.RegisterCreatedObjectUndo` 注册。

## 完整工作流：导出 → 跨场景 → 导入

```
场景 A 中配置好所有节点
  → Tools > Export Nodes to JSON
  → 生成 nodes.场景A.json
  → 切换到场景 B
  → Tools > Import Nodes from JSON
  → 选择 nodes.场景A.json
  → 冲突策略选择，确认
  → 节点以球体形式创建在 "Nodes" 父对象下
  → 微调 Transform / 字段（如有需要）
  → 运行验证
```
