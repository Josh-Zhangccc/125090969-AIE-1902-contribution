# 2026-05-03 更新：UI 触发交互重做

## 概述

本次更新重构了导览系统的触发-UI 交互流程。原来的逻辑是"靠近节点自动弹 UI"，不符合交互习惯；现在改为：

- **可见节点（POI 主节点 / 附属节点）**：靠近后不自动触发，显示"按 F 查看"提示 → 玩家按 F 键 → 弹出主内容面板（图片、正文、翻页） → ESC 关闭
- **不可见节点（隐形路径节点）**：靠近自动触发 → 弹出**底部字幕面板**（含播放/静音切换按钮） → 持续 N 秒后自动消失（N 由新增的 `autoDismissDuration` 字段控制），也可 ESC 提前关闭

## 功能详解

### 一、节点数据层：新增 `autoDismissDuration`

**NodeData.cs / NodeComponent.cs**

| 字段 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `autoDismissDuration` | float | 0 | 隐形节点 UI 自动消失的秒数。0 表示手动关闭（用于可见节点），> 0 表示 N 秒后自动关闭（用于隐形节点） |

在 `NodeComponent.OnValidate()` 中，当取消勾选 `Is Visible` 且 `autoDismissDuration` 为 0 时，会输出 Warning 提示开发者设置该值。

---

### 二、触发层：approach / trigger 分离

**ProximityTrigger.cs**

这是本次改动最核心的部分。原来 `Check()` 方法对所有节点一视同仁：进入半径 → Invalidate → OnNodeEnter。现在根据 `isVisible` 分流：

```
进入节点半径
    │
    ├─ isVisible == true  →  设为 "approached" 状态
    │                        ├─ 触发 OnNodeApproached（显示"按 F 查看"）
    │                        ├─ 不 Invalidate（离开后还可再靠近）
    │                        └─ 等待玩家按 F → TriggerApproachedNode()
    │                            └─ Invalidate + OnNodeEnter → 正常内容加载
    │
    └─ isVisible == false →  保持原有行为
                             ├─ Invalidate 立即执行
                             └─ OnNodeEnter → 内容加载 → 底部字幕面板
```

**新增公开成员：**

| 成员 | 类型 | 说明 |
|------|------|------|
| `OnNodeApproached` | event | 进入可见节点范围（未触发）时触发 |
| `OnApproachCleared` | event | 离开可见节点范围（未触发就走了）时触发 |
| `ApproachedNodeComponent` | property | 当前靠近但未触发的可见节点 |
| `HasApproachedNode` | property | 是否有可见节点处于 approached 状态 |
| `TriggerApproachedNode()` | method | 手动触发当前 approached 的节点（由 F 键调用） |

**状态转换逻辑：**

```
None ──进入可见节点──→ Approached ──按F──→ Triggered ──离开──→ None
  │                      │                    │
  │                      │ 离开（不走F）         │ 离开
  │                      ↓                    ↓
  │                     None                  None
  │
  └──进入不可见节点──→ Triggered（自动）
```

### 四、UI 层：双面板 + F 键 + 自动关闭

**ContentDisplay.cs**

**新增 Inspector 字段：**

| 区域 | 字段 | 类型 | 说明 |
|------|------|------|------|
| **Trigger** | `triggerKey` | Key | 触发可见节点的按键，默认 F |
| **Prompt** | `promptText` | TextMeshProUGUI | 靠近可见节点时的提示文字（如"按 F 查看"） |
| **Invisible Node Panel** | `invisibleContentPanel` | GameObject | 隐形节点底部字幕面板根节点 |
| | `invisibleTitleText` | TextMeshProUGUI | 字幕标题 |
| | `invisibleBodyText` | TextMeshProUGUI | 字幕正文 |
| **Play/Mute** | `playMuteButton` | Button | 播放/静音切换按钮 |
| | `playMuteButtonLabel` | TextMeshProUGUI | 按钮文字标签 |

**面板路由逻辑（OnContentReady）：**

- `data.isVisible == true` → 显示主内容面板 `contentPanel`（图片、正文、翻页），隐藏字幕面板
- `data.isVisible == false` → 显示底部字幕面板 `invisibleContentPanel`，隐藏主内容面板

**播放/静音按钮：**

单按钮切换逻辑：
- 音频播放中 → 显示"🔇 静音"，点击后暂停音频，切换为"▶ 播放"
- 音频已暂停 → 显示"▶ 播放"，点击后恢复播放，切换为"🔇 静音"

**自动关闭协程：**

当 `data.autoDismissDuration > 0` 时，启动一个协程等待 N 秒后自动调用 `Dismiss()`。ESC 键可提前关闭（Dismiss 会取消协程）。

## 完整工作流

### 可见节点流程

```
玩家靠近可见节点
  → ProximityTrigger.Check() 检测到
  → isVisible=true → isApproach=true
  → 触发 OnNodeApproached
  → ContentManager 转发 OnNodeApproached
  → ContentDisplay 显示 promptText："按 F 查看"
  → 玩家按 F
  → ContentDisplay.OnTriggerAction()
  → ContentManager.TriggerPendingNode()
  → ProximityTrigger.TriggerApproachedNode()
      → Invalidate + OnNodeEnter
  → ContentManager 加载内容（图片/文字/音频）
  → ContentDisplay 显示主内容面板 contentPanel
  → 玩家浏览内容（翻页、看图片）
  → ESC → Dismiss() → 面板关闭
```

### 不可见节点流程

```
玩家靠近不可见节点
  → ProximityTrigger.Check() 检测到
  → isVisible=false → isApproach=false
  → Invalidate + OnNodeEnter
  → ContentManager 加载内容（文字/音频）
  → ContentDisplay 显示底部字幕面板 invisibleContentPanel
  → 音频自动播放，按钮显示"🔇 静音"
  → autoDismissDuration 秒后自动调用 Dismiss()
      （或 ESC 提前关闭，或点击播放/静音切换）
```

## Unity 场景搭建指南

以下操作按 **Node（节点） → Trigger（触发） → UI（界面）** 三部分组织。假设场景中已有 Player 和基本地形。

### Part 1 — Node 节点搭建

#### 1.1 创建可见节点（POI）

```
Hierarchy 右键 → Create Empty → 命名 "POI_xxx"
Add Component → NodeComponent
```

Inspector 中设置：

| 区域 | 字段 | 值 |
|------|------|-----|
| **Identity** | Id | `myPOI`（唯一标识，小驼峰） |
| | Display Name → Zh | `测试节点` |
| | Display Name → En | `Test Node` |
| | Short Description → Zh | `这是一个测试节点` |
| | Short Description → En | `This is a test node` |
| | Is Visible | **勾选** |
| **Trigger** | Radius | `5` |
| | Auto Dismiss Duration | `0`（可见节点无需自动消失） |
| **Media** | Zh Text → Title | `测试标题` |
| | Zh Text → File | `Content/Text/test.zh.md` |
| | En Text → Title | `Test Title` |
| | En Text → File | `Content/Text/test.en.md` |

拖动 GameObject 到你想要的位置（Inspector 中 Position 会实时更新）。

#### 1.2 创建不可见节点（路径节点）

```
Hierarchy 右键 → Create Empty → 命名 "Path_xxx"
Add Component → NodeComponent
```

Inspector 中设置：

| 区域 | 字段 | 值 |
|------|------|-----|

| | Is Visible | **取消勾选** |

| | En Text → Title | `Path Narration` |

> 不可见节点的 Display Name 和 Short Description 可留空。建议使用 `path_` 前缀命名。

#### 1.3 准备文字内容文件

在 `Assets/StreamingAssets/Content/Text/` 下创建 Markdown 文件，例如 `test.zh.md`：

同理创建对应的 `.en.md` 英文文件。

### Part 2 — Trigger 触发搭建

#### 2.1 确认 Player 上有 ProximityTrigger

选中 Player GameObject，确认 Inspector 中有 `ProximityTrigger` 组件。如果没有：

```
Add Component → ProximityTrigger
```

Inspector 设置：

| 字段 | 值 | 说明 |
|------|-----|------|
| Check Interval | `0.25` | 检测间隔（秒），越小响应越快 |
| Enable Debug Log | 勾选 | 在 Console 输出触发日志，调试完成后可关闭 |

#### 2.2 创建 TourSession 管理对象

```
Hierarchy 右键 → Create Empty → 命名 "TourSession"
Add Component → TourSessionManager
Add Component → TriggerSystemDebugger（可选，方便调试）
```

TriggerSystemDebugger 的 `Show Overlay` 勾选后可在 Game 视图左上角看到已访问节点数。

#### 2.3 创建 NodeRegistry

```
Hierarchy 右键 → Create Empty → 命名 "NodeRegistry"
Add Component → NodeRegistry
```

运行场景后，所有 NodeComponent 会自动注册到 NodeRegistry。

### Part 3 — UI 界面搭建

#### 3.1 创建 Canvas

```
Hierarchy 右键 → UI → Canvas
```

自动生成 Canvas + EventSystem。

#### 3.2 创建主内容面板（可见节点使用，沿用已有面板）

在 Canvas 下创建 Panel，命名 `ContentPanel`。**默认不激活**（Inspector 左上角取消勾选）。

内含子组件：

```
ContentPanel                            ← Panel（Image 做背景）
├── TitleText                           ← TextMeshProUGUI（标题，加粗 28pt）
├── BodyText                            ← TextMeshProUGUI（正文，多行 20pt）
├── MainImage                           ← Image（内容图片）
├── ImageDescText                       ← TextMeshProUGUI（图片说明）
├── PageIndicator                       ← TextMeshProUGUI（"1 / 3"）
├── PrevImageButton                     ← Button - TextMeshPro（上一张）
├── NextImageButton                     ← Button - TextMeshPro（下一张）
├── PlayAudioButton                     ← Button - TextMeshPro（播放音频，可选）
├── PauseAudioButton                    ← Button - TextMeshPro（暂停音频，可选）
└── CloseButton                         ← Button - TextMeshPro（关闭按钮，可选）
```

#### 3.3 创建靠近提示文字

在 **Canvas 直接子级**（不要放在 ContentPanel 下面）创建：

```
Hierarchy 右键 → UI → Text - TextMeshPro → 命名 "PromptText"
```

> 关键：PromptText 必须在 Canvas 直接子级，不能是 ContentPanel 的子对象，否则会因为父级 inactive 而无法显示。

设置：

- 位置：屏幕居中偏下（Pos Y ≈ -100）
- 字体大小：28-32pt
- 颜色：淡黄色或白色
- 勾选激活状态（初始激活，代码会在不需要时隐藏）

#### 3.4 创建底部字幕面板（不可见节点使用）

在 **Canvas 直接子级** 创建：

```
Hierarchy 右键 → Create Empty → 命名 "InvisibleContentPanel"
Add Component → Image（黑色半透明背景 RGBA: 0,0,0,180）
```

设置 Rect Transform：

- Anchor Preset → bottom-stretch（底部拉伸，高度约 80-100px）
- 默认**不激活**（Inspector 左上角取消勾选）

在其下创建子组件：

```
InvisibleContentPanel
├── InvisibleTitleText                  ← TextMeshProUGUI（标题，22pt，左对齐）
├── InvisibleBodyText                   ← TextMeshProUGUI（正文，18pt，浅灰色）
└── PlayMuteButton                      ← Button - TextMeshPro（右侧）
    └── ButtonLabel                     ← TextMeshProUGUI（初始 "🔇 静音"）
```

## 运行时行为速查表

| 场景 | 预期行为 |
|------|----------|
| 靠近可见节点 | promptText 出现"按 F 查看"，主面板不弹 |
| 按 F | 主面板弹出（图片+正文+翻页），promptText 消失 |
| 主面板显示时按 ESC | 面板关闭，音频停止 |
| 离开可见节点（未按 F） | promptText 消失 |
| 靠近不可见节点（autoDismissDuration=5） | 底部字幕面板弹出，5 秒后自动消失 |
| 字幕面板显示时按 ESC | 立即关闭，计时器取消 |
| 字幕面板显示时点播放/静音按钮 | 音频暂停/恢复，按钮文字切换 |
| 再次靠近已触发的可见节点 | 无反应（已 invalidated） |

## 调试技巧

1. **Console 日志**：
   - `[ProximityTrigger] Approached: xxx` — 靠近可见节点
   - `[ProximityTrigger] Triggered by user: xxx` — F 键触发
   - `[ProximityTrigger] Approach cleared: xxx` — 离开可见节点
   - `[ContentManager] 内容加载完成` — 内容就绪

2. **TriggerSystemDebugger Overlay**：勾选 `Show Overlay` 可在 Game 视图看到 Visited 计数。

3. **Scene View Gizmos**：选中 Player，绿色线表示在某个节点半径内，灰色线表示最近节点在范围外。

4. **Tools 菜单**：
   - `Tools > Tour System > Validate All Nodes` — 检查所有节点配置
   - `Tools > Tour System > Export Nodes to JSON` — 导出 nodes.json

5. **如果 promptText 没出现**：检查它是否是 ContentPanel 的子对象（必须是 Canvas 直接子级），以及 ContentDisplay 的 Prompt Text 槽位是否已绑定。
